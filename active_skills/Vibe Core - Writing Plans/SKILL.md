# Vibe Core: Writing Plans

来源：Vibe-Skills `core/skills/writing-plans`

## 触发场景
- 需求已明确，准备进入实施
- 需要把设计转成可执行的工程计划
- 需要交接给其他人/下个会话继续推进

## 推荐工作流
1. 确认范围与约束已冻结
2. 标出会改到的文件或模块路径
3. 拆成小步、可验证的 action items
4. 为每步补上验证命令或验收方式
5. 保证计划对低上下文接手者也清晰可执行

## 协同建议
- 需求还没清楚：先用 `Vibe Core - Brainstorming`
- 需要快速计划草案：配合 `Vibe Theme - Create Plan`
- 涉及系统设计：配合 `Vibe Theme - AIOS Architect`
- 准备分治推进：配合 `Vibe Core - Subagent Development`

## 不适合
- 需求边界还没冻结
- 用户只是想要一句话建议，不需要工程计划
